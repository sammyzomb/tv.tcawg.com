專案已連到 GitHub：`https://github.com/sammyzomb/tv.tcawg.com.git`

## Netlify 發佈（你現在用的網頁空間）

Netlify 是連到同一個 GitHub repo，**push 到指定分支**就會自動建置並發佈。

### 讓最新改動上線（導覽列、hero、下拉半透明）

你的改動在分支 **feat-add-loading-ui-880d1**，要讓 Netlify 更新，需合併到 **main** 再 push：

1. **在 GitHub 開 Pull request**
   - 打開：https://github.com/sammyzomb/tv.tcawg.com
   - 分支選 **feat-add-loading-ui-880d1**，點 **Compare & pull request**
   - Base 維持 **main**，建立 PR 後按 **Merge pull request** → **Confirm merge**

2. **或在本機合併後 push**
   ```bash
   git checkout main
   git pull origin main
   git merge feat-add-loading-ui-880d1
   git push origin main
   ```

3. **到 Netlify 看結果**
   - 登入 **https://app.netlify.com**
   - 點你的 site（可能叫 tv.tcawg.com 或類似名稱）
   - **Deploys** 會顯示最新部署；完成後你的網址就是新版本

### 查你的 Netlify 網址

- **Site overview** 上方會寫：`https://xxxxx.netlify.app` 或已綁定的 **Custom domain**（例如 **https://tv.tcawg.com**）
- **Domain settings** 可看目前用的是 Netlify 子網域還是自訂網域

### 要部署哪個分支

- **Site configuration** → **Build & deploy** → **Continuous deployment** → **Branch to deploy**
- 一般是 **main**；若你設成 **feat-add-loading-ui-880d1**，該分支 push 也會觸發部署

## 方式二：GitHub Pages（備用／同 repo）

1. **先推送到 GitHub**
   - 在專案資料夾 `teavel-video-site` 開啟終端機，執行：
   ```bash
   git status
   git add .
   git commit -m "發佈：導覽列與 hero 調整、下拉半透明"
   git push origin main
   ```

2. **在 GitHub 開啟 Pages**
   - 打開：https://github.com/sammyzomb/tv.tcawg.com
   - 上方選單 **Settings** → 左側 **Pages**
   - **Source** 選 **Deploy from a branch**
   - **Branch** 選 `main`，資料夾選 **/ (root)**，按 **Save**

3. **等幾分鐘**
   - 完成後網址會是：**https://sammyzomb.github.io/tv.tcawg.com/**  
   - 若 repo 名是 `tv.tcawg.com`，也可能是：**https://sammyzomb.github.io/tv.tcawg.com/**

## 方式二：Netlify（拖曳資料夾即可）

1. 打開 **https://app.netlify.com**（可免費註冊）
2. 把整個 **teavel-video-site** 資料夾拖進 Netlify 的「Deploy」區
3. 部署完成會給一個隨機網址（例如 `xxx.netlify.app`），可再綁自己的網域

---

## 注意

- **header.html** 是 iframe 載入的，需與 index 同網域，用上述任一種方式發佈整包即可，不需額外設定。
- 若有用 **Contentful / API 金鑰**，確認 `browser-config.js` 不要提交敏感資訊；正式環境可改用 Netlify/ Vercel 的環境變數。
- 發佈後若導覽列或影片有異常，用瀏覽器開發者工具 (F12) 看 Console 是否有跨域或載入錯誤。
